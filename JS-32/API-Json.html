Wanneer welk Ajax request te gebruiken?

  1. AJAX-requests in Plain Javascript worden waarschijnlijk alleen nog gebruikt door code-freaks die alles in plain Javascript willen blijven schrijven en geen libraries gebruiken.
    Met de komst van de FETCH API die met Javascript meegeleverd wordt hoeft niet eens meer een aparte library geladen te worden.. Daarnaast is de request in Plan Javascript lastiger te schrijven en heeft veel regels nodig.
    Waarom moeilijk doen als het makkelijker kan...
  2. AJAX-requests met de FETCH API van Javascript zelf zijn eenvoudig te schrijven en er is geen aparte include van een library in nodig. De FETCH API is qua gebruikersgemak erg vergelijkbaar met Axios.
    Lees dit artikel over de verschillen tussen de FETCH API en Axios.
  3. AJAX-requests met Axios zijn eenvoudig te schrijven maar je moet wel eerst de Axios libray laden of includen. Qua gebruikersgemak erg vergelijkbaar met de FETCH API.
    Lees dit artikel over de verschillen tussen de FETCH API en Axios.
  4. AJAX-requests met behulp van jQuery lijken qua gebruikersgemak veel op de de FETCH API en Axios.
    Voor jQuery moet je wel een library laden of includen waarvan de AJAX-requests maar een heel klein gedeelte zijn. Dus een te zware library voor alleen de AJAX-requests. Gebruik je sowieso al jQuery in de site dan zou het een logisch keus zijn om ook de AJAX-requests van jQuery te gebruiken.
